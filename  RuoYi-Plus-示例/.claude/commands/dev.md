# /dev - 智能开发命令

## 描述
按照7步标准流程开发新功能,确保代码质量和规范性

## 使用方法
```
/dev 开发用户管理功能
/dev 实现优惠券模块
/dev 添加订单导出功能
```

## 执行步骤

### 步骤 1: 需求澄清
- [ ] 理解用户要实现的功能
- [ ] 主动询问以下问题:
  - 功能的业务背景是什么?
  - 涉及哪些实体对象?
  - 需要哪些字段和属性?
  - 是否需要权限控制?
  - 是否需要前端页面?

### 步骤 2: 技术设计
- [ ] 确定涉及的数据库表
- [ ] 设计 API 接口 (遵循 RESTful 规范)
- [ ] 规划前端页面和组件
- [ ] 列出需要激活的技能:
  - `crud-development` - CRUD开发
  - `database-ops` - 数据库操作
  - `api-development` - API设计
  - `ui-pc` - 前端开发 (如果需要)

### 步骤 3: 数据库设计
- [ ] 编写建表 SQL (参考 `database-ops` 技能)
- [ ] 配置数据字典
- [ ] 配置菜单和权限
- [ ] 插入测试数据 (可选)

### 步骤 4: 后端开发
按照四层架构生成代码:
- [ ] Entity (实体类)
- [ ] BO (业务对象)
- [ ] VO (视图对象)
- [ ] DAO (数据访问对象)
- [ ] Mapper (MyBatis映射)
- [ ] Service (服务层)
- [ ] Controller (控制器)

### 步骤 5: 前端开发 (如果需要)
- [ ] 生成 API 定义 (`api/xxx.ts`)
- [ ] 生成类型定义 (`types/xxx.ts`)
- [ ] 生成页面组件 (`views/xxx/index.vue`)
- [ ] 配置路由 (`router/index.ts`)
- [ ] 配置权限 (`permission.ts`)

### 步骤 6: 测试验证
- [ ] 启动后端服务
- [ ] 启动前端服务
- [ ] 执行功能测试
- [ ] 验证权限控制
- [ ] 验证数据字典
- [ ] 验证异常处理

### 步骤 7: 文档更新
- [ ] 更新项目进度 (`docs/项目状态.md`)
- [ ] 记录待办事项 (`docs/待办清单.md`)
- [ ] 更新 API 文档
- [ ] 提交代码到 Git

## 示例执行

用户输入:
```
/dev 开发优惠券管理功能
```

AI 执行流程:
```
### 需求澄清
我需要了解一些细节:
1. 优惠券的类型有哪些? (满减券、折扣券、立减券)
2. 需要哪些字段? (名称、类型、金额、数量、有效期等)
3. 是否需要审批流程?
4. 是否需要前端管理页面?

### 技术设计
涉及表: bus_coupon
激活技能: crud-development, database-ops

### 数据库设计
[生成建表SQL、字典SQL、菜单SQL]

### 后端开发
[生成 Entity、BO、VO、DAO、Service、Controller]

### 前端开发
[生成 API、页面组件]

### 测试验证
[执行测试]

### 文档更新
[更新项目状态]
```

## 注意事项
1. 每个步骤完成后等待用户确认再继续
2. 遵循项目规范 (参考 SKILL.md)
3. 使用统一的返回格式和异常处理
4. 所有代码必须有注释
5. 提交代码前使用 @code-reviewer 审查

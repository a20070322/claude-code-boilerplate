# /dev - 智能开发命令

## 描述
按照7步标准流程开发新功能,确保代码质量和规范性

## 使用方法
```
/dev 开发用户管理功能
/dev 实现优惠券模块
/dev 添加订单导出功能
```

## 执行步骤

### 步骤 1: 需求澄清
- [ ] 理解用户要实现的功能
- [ ] 主动询问以下问题:
  - 功能的业务背景是什么?
  - 涉及哪些实体对象?
  - 需要哪些字段和属性?
  - 是否需要权限控制?
  - 是否需要前端页面?

### 步骤 2: 技术设计
- [ ] 确定涉及的数据模型
- [ ] 设计 API 接口 (遵循 RESTful 规范)
- [ ] 规划前端页面和组件
- [ ] 列出需要激活的技能

### 步骤 3: 数据/模型设计
- [ ] 设计数据模型
- [ ] 设计数据库表结构 (如果需要)
- [ ] 配置相关资源
- [ ] 插入测试数据 (可选)

### 步骤 4: 后端/核心开发
- [ ] 实现数据模型
- [ ] 实现数据访问层
- [ ] 实现服务层
- [ ] 实现控制器/API

### 步骤 5: 前端开发 (如果需要)
- [ ] 生成 API 定义
- [ ] 生成类型定义
- [ ] 生成页面组件
- [ ] 配置路由
- [ ] 配置权限 (如果需要)

### 步骤 6: 测试验证
- [ ] 启动后端服务
- [ ] 启动前端服务 (如果需要)
- [ ] 执行功能测试
- [ ] 验证权限控制
- [ ] 验证异常处理

### 步骤 7: 文档更新
- [ ] 更新项目进度
- [ ] 记录待办事项
- [ ] 更新 API 文档
- [ ] 提交代码到 Git

## 注意事项
1. 每个步骤完成后等待用户确认再继续
2. 遵循项目规范 (参考 code-patterns 技能)
3. 使用统一的返回格式和异常处理
4. 所有代码必须有注释
5. 提交代码前进行代码审查

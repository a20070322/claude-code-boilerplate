# 项目探索技能（老项目学习）

## 触发条件
- 关键词: 项目结构、探索、了解、学习
- 场景: 接手新项目或需要理解现有项目时

## 探索流程

### 1. 项目结构分析
**扫描目录:**
- 识别主要目录（src, lib, app, pages, components 等）
- 识别配置文件（package.json, pom.xml, requirements.txt 等）
- 识别构建文件（webpack, vite, gradle 等）

**输出:**
```
## 项目结构
├── src/          # 源代码
├── config/       # 配置文件
├── tests/        # 测试文件
└── docs/         # 文档
```

### 2. 技术栈识别
**检测文件:**
- package.json → JavaScript/TypeScript
- pom.xml → Java/Maven
- requirements.txt → Python
- go.mod → Go

**识别框架:**
- React, Vue, Angular (前端)
- Spring Boot, Express, Django (后端)
- 数据库、缓存、消息队列

### 3. 核心模块识别
**查找:**
- 入口文件（index.js, App.java, main.go）
- 路由配置
- 状态管理
- 数据层

### 4. 依赖关系分析
**分析:**
- 外部依赖包
- 内部模块引用
- 关键依赖关系

### 5. 关键文件定位
**找出:**
- 配置文件
- 环境变量文件
- API 定义文件
- 类型定义文件

## 输出格式

```
# 项目探索报告

## 📁 项目结构
[目录树]

## 🔧 技术栈
- 语言: ...
- 框架: ...
- 构建工具: ...

## 🎯 核心模块
1. [模块名] - [说明]
2. [模块名] - [说明]

## 📦 依赖关系
- 主要依赖: ...
- 关键库: ...

## 📄 关键文件
- 入口: ...
- 配置: ...
- API: ...

## 💡 下一步建议
1. ...
2. ...
```

## 使用示例

用户: "帮我了解这个项目"

AI 执行:
1. 扫描项目结构
2. 识别技术栈
3. 分析核心模块
4. 生成探索报告

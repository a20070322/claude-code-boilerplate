# 文档开发技能

## 触发条件
- 关键词: 文档、README、指南、手册、说明
- 场景: 编写或修改项目文档时

## 文档类型

### 1. README 文档（项目主页）
**必需章节:**
```markdown
# 项目名称

简短描述 (1-2 句话)

## 简介
[项目的详细说明]

## 快速开始
```bash
# 步骤 1
command
```

## 功能特性
- ✅ 特性 1
- ✅ 特性 2

## 使用示例
```javascript
// 代码示例
```

## 相关链接
- 文档链接
```

### 2. 模板说明文档
**包含:**
- 技术栈
- 包含内容
- 快速开始
- 注意事项

### 3. CLAUDE.md 文档（项目规范）
**包含:**
- 项目概述
- 核心规范
- 快捷命令
- 技能列表
- 代理列表
- 开发流程

## 文档编写原则

### 1. 清晰优先
- 使用简洁的语言
- 避免技术术语堆砌
- 提供具体示例

### 2. 结构化
- 使用清晰的层级（h1 > h2 > h3）
- 合理的分节
- 代码块标注语言

### 3. 可维护性
- 统一的格式风格
- 版本更新记录
- 相关文档引用

## 检查清单
- [ ] 标题层级正确
- [ ] 代码块标注语言
- [ ] 链接可访问
- [ ] 示例代码准确
- [ ] 包含必要的步骤
- [ ] 使用列表和表格增强可读性
